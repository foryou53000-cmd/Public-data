[1] NEIS 학교기본정보 Open API

- 데이터 성격
  교육 공공데이터

- 제공/운영 출처
  NEIS Open API
  교육부 및 시도교육청 학교 정보 기반 공공데이터로 활용

- API 주소
  https://open.neis.go.kr/hub/schoolInfo

- 앱 코드 내 사용 위치
  학교 검색 함수 sch()
  URL 예시:
  https://open.neis.go.kr/hub/schoolInfo?KEY=...&Type=json&pIndex=1&pSize=15&SCHUL_NM=학교명

- 주요 사용 필드
  SCHUL_NM: 학교명
  SCHUL_KND_SC_NM: 학교급
  ORG_RDNMA: 학교 도로명 주소
  ORG_RDNDA: 학교 상세 주소
  JU_ORG_NM: 관할/지역 정보

- 앱에서의 활용 방식
  1. 사용자가 학교명을 검색하면 NEIS 학교기본정보 API에서 실제 학교 정보를 조회한다.
  2. 선택한 학교의 주소를 기준으로 학교 위치를 산출한다.
  3. 학교 주소에서 경북 시군을 추정해 같은 시군 관광자원에 지역화 보정을 적용한다.
  4. 학교와 관광자원 사이의 거리를 계산해 체험학습 가능성을 판단한다.

- 심사 대응 문장
  본 앱은 NEIS 학교기본정보 Open API를 활용하여 사용자의 실제 학교 정보를 조회하고,
  학교 위치를 기준으로 경북 관광자원과 교육과정 연계 추천을 수행한다.
  따라서 교육 공공데이터 1개 이상 활용 필수 요건을 충족한다.


[2] 2022 개정 사회과 교육과정 성취기준

- 데이터 성격
  교육과정 공식 기준 데이터
  현재 앱에서는 외부 API로 조회하지 않고, 앱 내부 교육과정 기준 데이터로 반영되어 있다.
  5, 6학년 2학기 교과서가 아직 없더라도 공식 성취기준을 기준으로 교육과정 연계가 가능하다.

- 제공/운영 출처
  교육부 고시 제2022-33호 [별책 7] 사회과 교육과정
  NCIC 국가교육과정정보센터 / 교육부 고시 원문

- 앱 코드 내 사용 위치
  CURRICULUM_SOURCE
  SOCIAL_CURRICULUM
  CUR[5].사회

- 주요 사용 항목
  성취기준 코드
  성취기준 문장
  역사 영역 단원 구조
  5학년 2학기 역사 단원:
    4. 유적과 유물로 살펴본 옛 사람들의 생활
    5. 달라지는 시대, 변화하는 생활 모습
    6. 식민 통치와 저항, 전쟁이 바꾼 사회와 생활

- 앱에서의 활용 방식
  1. 5학년 2학기 역사 단원은 교과서 단원명이 아니라 공식 성취기준을 최상위 기준으로 삼는다.
  2. 관광자원 후보를 성취기준 코드별 키워드와 비교한다.
  3. 팝업에 성취기준 코드, 추천 이유, 관련 주제·차시, 직접 연계 근거를 제시한다.
  4. 5, 6학년 2학기 교과서가 출간되면 단원명과 차시명을 보완한다.


[3] 국사편찬위원회 한국사데이터베이스 공공데이터

- 데이터 성격
  교육 공공데이터 후보 및 역사 근거 참고 자료
  국사편찬위원회는 기준표의 교육부 소속기관 목록에 포함된다.
  단, 현재 HTML 앱 코드에서 db.history.go.kr 또는 공공데이터포털 국사편찬위원회 API를 실시간 호출하지는 않는다.
  현재는 역사 단원 키워드, 시대 태그, 추천 이유 문장, 출처 표기 수준으로 반영되어 있다.

- 제공/운영 출처
  교육부 국사편찬위원회
  공공데이터포털(data.go.kr)
  한국사데이터베이스(db.history.go.kr)

- 앱 코드 내 사용 위치
  NIKH_SOURCE
  HISTORY_PUBLIC_SOURCE
  SOCIAL_CURRICULUM 5학년 2학기 역사 단원
  socialUnitProfile()
  recommendationReasonFor()
  directReasonFor()

- 현재 참고 근거로 반영한 주요 데이터셋
  교육부 국사편찬위원회_한국사데이터베이스 정보_삼국유사 원문
  교육부 국사편찬위원회_한국사데이터베이스 정보_한국고대금석문
  교육부 국사편찬위원회_한국사데이터베이스 정보_고려사 원문
  교육부 국사편찬위원회_한국사데이터베이스 정보_한국사료총서 원문
  교육부 국사편찬위원회_한국사데이터베이스 정보_일제감시대상인물카드
  교육부 국사편찬위원회_역사지리정보 배경지도

- 앱에서의 활용 방식
  1. 5학년 2학기 역사 단원별로 관련 사료 DB 성격을 시대·사건·장소 키워드로 태깅한다.
  2. 경북 관광자원의 장소명, 주소, 관광분류, 상세 설명을 사료 태그와 비교한다.
  3. 고대 단원은 삼국유사, 한국고대금석문, 신라·가야 유적과 연결한다.
  4. 고려 단원은 고려사 원문, 한국사료총서, 고려 불교 문화유산과 연결한다.
  5. 일제강점기와 전쟁 단원은 일제감시대상인물카드, 한국사료총서, 자료대한민국사 성격의 자료와 연결한다.
  6. 추천 장소 팝업에 "교육공공데이터 원자료 기반 추천 이유"가 드러나도록 표시한다.

- 현재 상태 주의
  직접 API 연결 완료가 아니라, 앱 내부 규칙과 설명 근거에 반영된 상태이다.
  심사 자료에서는 "국사편찬위원회 API 실시간 연동"이라고 쓰면 안 되고,
  "국사편찬위원회 한국사DB를 역사 단원 시대 태그와 연계 근거 설계에 참고"라고 표현해야 한다.

[3] 한국관광공사 Tour API / 공공데이터포털

- 데이터 성격
  관광 공공데이터
  교육 공공데이터는 아니지만, 교육과정 데이터와 결합되는 지역 관광자원 원자료이다.

- 제공 출처
  한국관광공사 Tour API
  공공데이터포털(data.go.kr)

- 앱 코드 내 기본 API 주소
  http://apis.data.go.kr/B551011/KorService2

- 앱에서 사용하는 주요 엔드포인트
  searchKeyword2: 단원 키워드 기반 관광자원 검색
  areaBasedList2: 경북 시군/관광유형 기반 관광자원 목록 조회
  locationBasedList2: 학교 위치 주변 관광자원 조회
  detailCommon2: 장소 상세 설명 overview 조회
  detailImage2: 장소 이미지 조회

- 주요 사용 필드
  contentid: 관광자원 고유 ID
  title: 장소명
  addr1: 주소
  mapx, mapy: 좌표
  cat1, cat2, cat3: 관광분류 코드
  contenttypeid: 관광자원 유형
  sigungucode: 시군 코드
  firstimage, firstimage2: 대표 이미지
  overview: 장소 상세 설명

- 앱에서의 활용 방식
  1. 경북 areaCode=35를 기준으로 관광자원 후보를 수집한다.
  2. contentid 기준으로 중복을 제거한다.
  3. 좌표가 없는 데이터는 지도 표시와 거리 계산이 어려우므로 제외한다.
  4. 장소명, 주소, 관광분류 코드, 상세 설명을 단원 키워드와 비교한다.
  5. 교육과정 적합성 점수를 통과한 장소만 추천 결과로 제공한다.
  6. 이미지와 상세 설명은 팝업 및 카드뉴스 구성에 활용한다.


[3-2] 국가유산청 전국 지정문화재 현황 Open API

- 데이터 성격
  역사·문화유산 공공데이터
  경북의 국가유산, 유적, 유물, 고대·고려·조선 관련 장소 후보를 확장하는 원자료이다.

- 제공 출처
  국가유산청
  공공데이터포털(data.go.kr)

- 앱 코드 내 기본 API 주소
  http://www.khs.go.kr/cha/SearchKindOpenapiList.do
  http://www.khs.go.kr/cha/SearchKindOpenapiDt.do

- 주요 사용 필드
  ccbaMnm1: 국가유산명
  ccmaName: 지정종목
  ccsiName: 시군구명
  ccbaKdcd, ccbaAsno, ccbaCtcd: 상세조회 키
  longitude, latitude: 좌표
  ccceName: 시대
  ccbaLcad: 소재지 상세
  imageUrl: 대표 이미지
  content: 국가유산 설명

- 앱에서의 활용 방식
  1. 경북 시도코드 ccbaCtcd=37을 기준으로 지정문화재 후보를 조회한다.
  2. 국보, 보물, 사적, 명승, 천연기념물, 민속문화유산, 시도유형문화유산, 시도기념물, 문화유산자료, 국가등록유산을 우선 수집한다.
  3. 단원 키워드로 국가유산명을 추가 검색해 Tour API 누락 장소를 보강한다.
  4. 좌표가 있는 국가유산만 지도와 거리 계산에 반영한다.
  5. Tour API 결과와 장소명+시군 기준으로 중복 제거한다.
  6. 상세 API의 시대, 분류, 설명을 팝업의 직접 연계 근거와 추천 이유에 활용한다.


[3-3] 국가보훈부 현충시설정보 API

- 데이터 성격
  독립운동·국가수호·전쟁 관련 공공데이터
  5학년 6단원 "식민 통치와 저항, 전쟁이 바꾼 사회와 생활" 추천 후보를 확장하는 핵심 원자료이다.

- 제공 출처
  국가보훈부

- 앱 코드 내 기본 API 주소
  http://mfis.mpva.go.kr/api/json/allSearch.do

- 주요 사용 필드
  facilnm: 시설명
  mgmtno: 관리번호
  addr: 주소
  xindex, yindex: 좌표
  gubun: 독립운동/국가수호 등 구분
  topic: 3·1운동, 6·25전쟁 등 주제
  indicate: 사건연도 또는 관련연도
  facilgubun: 기념관, 비석, 탑 등 시설구분
  facilexplain: 건립취지/시설설명

- 앱에서의 활용 방식
  1. 현충시설 전체 목록을 조회한 뒤 경북 주소만 필터링한다.
  2. 독립운동, 의병, 항일, 일제, 3·1운동, 광복, 6·25전쟁, 호국, 전적 등 5학년 6단원 핵심 키워드와 비교한다.
  3. 좌표가 있는 시설만 지도와 거리 계산에 반영한다.
  4. Tour API 결과와 장소명+시군 기준으로 중복 제거한다.
  5. 팝업에는 국가보훈부 원자료의 구분, 주제, 연도, 설명을 직접 연계 근거로 활용한다.

- 앱에서의 활용 방식
  추천 관광자원과 학교 위치를 지도 위에 표시한다.

- 주의점
  심사에서 공공데이터 활용 근거로 제시하기보다는 서비스 구현 보조 도구로 설명한다.

[6] 단원별 선별 기준 데이터

- 데이터 성격
  교과서 분석 결과를 앱 내부 규칙으로 변환한 파생 데이터

- 앱 코드 내 사용 위치
  socialUnitProfile()
  curriculumRelevance()

  curriculumData()

- 포함 항목
  단원별 강한 근거 키워드
  단원별 보조 키워드
  단원별 제외 패턴
  단원별 최소 통과 점수
  관광 API 분류 코드 그룹
  세부 주제/차시 분기 규칙

- 앱에서의 활용 방식
  1. 관광자원이 단원과 직접 관련되는지 먼저 판정한다.
  2. 강한 근거 키워드, 보조 키워드, 관광분류 코드, 세부 주제 점수를 합산한다.
  3. 단원별 최소 통과 점수에 미달하면 추천에서 제외한다.
  4. 예를 들어 5학년 1단원 '우리나라 국토 여행'은 지형, 해안, 울릉, 독도, 지질 관련 근거가 있어야 통과한다.
  5. 문화원, 시장, 공방 등 단원과 직접 관련이 부족한 장소는 해당 단원 추천에서 제외한다.

  장소 추천 결과에 대한 사용자 반응을 브라우저에 저장한다.

- 주의점
